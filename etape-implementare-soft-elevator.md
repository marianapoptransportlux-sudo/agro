# Etape Implementare Soft Elevator

## Varianta aleasa

- Varianta aleasa pentru implementare este impartirea pe module de business.
- Scopul este dezvoltarea sistemului in ordinea naturala a fluxului real de lucru din elevator.

## Etapa 1 - fundatie si configurare

- nomenclatoare si setari de baza
- utilizatori, roluri si drepturi
- parteneri
- produse
- cilindri / locatii de stoc
- tarife si reguli de calcul

## Etapa 2 - receptie si intrare in stoc

- receptie
- cantarire manuala
- calitate: umiditate si impuritati
- calcul preliminar de plata
- intrare in stoc

## Etapa 3 - procesare si ajustari

- procesare
- curatire
- uscare
- deseuri
- ajustare la cantitatea finala
- transfer intern intre locatii

## Etapa 4 - livrare si reclamatii

- livrare / iesire
- scadere din stoc
- legare la factura / invoice
- reclamatii / neajunsuri

## Etapa 5 - plati si incasari

- plati catre furnizori
- incasari de la cumparatori
- stari financiare:
  - neachitat / partial achitat / achitat
  - neincasat / partial incasat / incasat
- jurnale financiare simple

## Etapa 6 - rapoarte, audit si notificari

- rapoarte zilnice
- raport procesare
- raport conducere
- audit / istoric modificari
- notificari Telegram bot

## Observatie

- Aceasta structura este aleasa pentru ca permite validare progresiva cu businessul, pe fluxuri reale, fara a construi prematur functii secundare.
